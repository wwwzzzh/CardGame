# CardGame - 纸牌消除游戏

用 Cocos2d-x 3.17 完成的一个纸牌消除小游戏。

---

## 环境

- Cocos2d-x 3.17
- Visual Studio 2022
- 分辨率 1080x2080 竖屏

## 怎么玩

游戏开始的时候，屏幕底下有一张底牌（翻开的）。上面有四摞牌，每摞四张，只有最底下那张是翻开的。

点翻开的牌，如果它的数字和底牌相差 1（比如底牌是 7，点 6 或 8），就匹配成功，这张牌会移到底牌位置变成新的底牌。花色不重要，只看数字。

每摞牌的覆盖关系是：最下面的正面牌压着上面三张背面牌。把正面牌消除之后，它上面的牌会自动翻正。

如果桌面上所有翻开的牌都无法与当前底牌匹配，点左边那叠备用牌，会翻一张新牌替换底牌。（只有当前无可匹配时才能抽备用牌。）

### 结束条件

- 上面 16 张牌全消完了 -> 屏幕中间弹出黄色 SUCCESS!（成功后不可操作）
- 备用牌也用完了，当前底牌又匹配不了任何一张牌 -> 黑色 DEFEAT!
  - 失败之后可以点 UNDO 回退，继续游戏

### 操作

| 操作 | 作用 |
|------|------|
| 点翻开的牌 | 和底牌匹配 |
| 点备用牌堆 | 抽一张新牌（仅当桌面无可匹配时可用） |
| 点 UNDO | 回退上一步 |
| 点 RESTART | 重新开始游戏 |

## 项目结构

```
proj.win32/
├── CardGame.sln              # 解决方案
├── CardGame.vcxproj          # 项目文件
├── main.cpp                  # 入口
├── AppDelegate.cpp/h         # 应用初始化
├── GameScene.cpp/h           # 游戏主逻辑
├── CardModel.cpp/h           # 数据和规则
├── CardSprite.cpp/h          # 卡牌渲染
Resources/
└── res/                      # 图片资源
```

简单说就是 MVC：

- CardModel 管数据和规则，不涉及任何界面
- CardSprite 管卡牌长什么样，不参与游戏逻辑
- GameScene 在上面两个中间搭桥，处理点击事件和动画

### 类结构

```
Sprite (cocos2d的)
  └── CardSprite               ← 基类，管触摸、翻牌动画、覆盖关系
         │  虚函数: loadFace() 
         │
         └── NormalCardSprite  ← 普通牌，加载数字和花色图片
```

### 可拓展

  想加新类型的牌（比如万能牌），直接从 CardSprite 继承，重写 loadFace() 就行，不用改其他文件。

## 一些实现细节

### 翻牌动画

从备用牌堆抽牌的时候，牌会在原地先翻面再移过去：

```
ScaleTo(0.15s, 缩到看不见) → 切换正反面 → ScaleTo(0.15s, 展开) → MoveTo(0.3s, 底牌位置)
```

### 回退

每执行一次操作，都把操作前的状态压到栈里。回退的时候弹出来恢复。回退时考虑了两种情况：

- 从桌面消除的牌 -> 找到原来位置，移回去重新变成正面
- 从牌堆抽的牌 -> 反向移回牌堆，翻回背面

失败（DEFEAT）后也可以使用 UNDO 回退一步继续游戏。

### 防抖

加了 _isAnimating 锁，动画没做完的时候不能点其他牌。还加了个 2 秒超时，防止锁卡死。

### 重新开始

任何时候点击 RESTART 按钮，都会创建一个全新的 GameScene 替换当前场景，重置所有状态开始新游戏。

### 备用牌抽取规则

只有当桌面上所有翻开的牌都无法与当前底牌匹配时，才允许从备用牌堆抽牌。这样可以防止玩家跳过可匹配的牌，保证游戏的策略性。
